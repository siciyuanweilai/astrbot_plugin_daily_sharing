## 📖 序言：让 Bot 在沉默之外开口

这是一款为 AstrBot 打造的每日主动分享插件。它不再只等待一句问题才回应，而是会循着时间、天气、生活状态、聊天记忆和网络热搜，在合适的时刻把所见所闻分享到你身边。

清晨可以是一句带着阳光的早安语音，午后可以是一张热搜长图，深夜可以是一段安静的心情视频，也可以自动写成 QQ 空间里的日常说说。它关心的不只是“分享出去”，而是让每一次主动分享都更自然、更像正在生活。


## 🌟 核心能力 

### 1. 🔍 真实来源：Web 联网搜索增强 (Tavily / Brave)
*   **拒绝热搜标题党**：优先使用新闻源 API 自带的摘要/正文；缺少背景时再自动接入全网检索能力（智能打通 AstrBot 配置，无缝切换 Tavily 或 Brave ）。分享新闻不再仅凭标题进行空洞脑补。
*   **API v2 与脏响应兼容**：热搜接口使用新版 `/api/v2/*` 路径，响应里即使混入调试标记、状态说明或 BOM，也会优先提取第一个有效 JSON 继续解析。
*   **百科+网页并发校验**：在进行【知识】与【推荐】时，并发调用“百度百科 + Web 搜索”。拥有扎实的数据支撑，从根本上解决 LLM 胡编乱造的幻觉问题。

### 2. 🌐 跨域分享：QQ 空间与独立早报联动 
*   **说说自动机**：无需人工干预，Bot会根据一天中的不同时段（早晨问候/深夜emo），自动以“第一人称”口吻生成配图说说并分享到QQ空间。
*   **登录态自动保活**：QQ空间 Cookie 过期或图片上传接口返回登录失效时，会尝试唤醒底层空间插件刷新登录态并自动重试分享，降低定时说说中断概率。
*   **早报速递员**：支持将【每天 60s 读懂世界】和【AI 资讯快报】作为独立定时任务，自动推送到群聊、私聊并分享至QQ空间。

### 3. 🧠 懂得分寸：群聊气氛与动态记忆
*   **KY 规避 (读空气)**：群聊态势感知。当群友正在热火朝天地讨论时，Bot会保持安静，绝不打断施法。
*   **记忆回写 (拒绝断片)**：Bot发送的配图都会被**写回AstrBot的对话历史**。你问：“你刚才发的照片是在哪拍的？”，它能对答如流。
*   **第二人称防呆**：深度优化底层提示词，彻底解决Bot在私聊时把用户当“第三人称”称呼的语病，交互更像真人伴侣。

### 4. 🎨 视觉与影像：形象参考与视频生成
*   **形象一致性**：联动新版**Gitee**生图插件，支持**形象参考图**。Bot发的生活照终于可以拥有稳定的“脸”和气质了！
*   **图文转视频**：一键将生成的静态生活照转换为**高清动态视频**。清晨的早安，是真的在对你挥手微笑。
*   **视频提示词**：视频生成会基于配图描述和分享文案分别设计镜头动作、人物微表情、环境声与人声氛围，尽量保持原图人物、服装、构图和场景一致。
*   **构图顾问**：内置**Agent**智能判断——分享心情时“画人”，分享新闻时“画景”。
*   **环境光影**：自动根据物理时间（清晨/黄昏/深夜）调整画面的光影，杜绝 "半夜窗外是大太阳" 的违和感。

### 5. 📰 见闻来源：全网热搜与自定义内容库
*   **众多热搜源**：微博、知乎、B站、抖音、头条、百度、腾讯、小红书、夸克、36氪、51CTO、A站、爱范儿、网易、新浪、澎湃、第一财经、财联社。
*   **长图原文反查**：新闻热搜长图会同步缓存同源 JSON，之后可用自然语言说“把刚才热搜图第三条原文发我”，由 LLM 调用工具取回对应新闻链接，并清理多余参考链接。
*   **时段偏好算法**：早晨看知乎/小红书，傍晚刷 B 站/微博，深夜看抖音，完美契合人类作息。
*   **专属内容库**：支持在配置项中完全自定义【知识】和【推荐】库的话题方向，并可选择是否显示“知识类型/推荐类型”前缀。

### 6. 🛡️ 稳定送达：双重降级机制
*   **LLM 自动降级**：指定的大模型 API 挂了？超时了？触发敏感词了？插件会自动无缝切换到系统默认的备用模型，确保分享从不缺席。
*   **新闻源熔断**：主新闻源获取失败时，自动在备选池中切换；全部失败时明确取消本次新闻分享并写入失败记录，避免输出无依据新闻。
*   **图片发送稳态链路**：群聊/私聊图片会优先下载到本地临时目录后再发送，规避不稳定 CDN 链接导致的 `rich media transfer failed`。
*   **临时文件自清理**：普通下载图和个人微信压缩副本统一进入 `Temp` 管理；个人微信压缩副本默认只保留最新 10 张，避免长期堆积。

### 7. ⏰ 灵活调度：定时策略与断点恢复
*   **时段序列**：不同时间段执行不同的逻辑（早晨发问候，下午发推荐，晚上发心情）。
*   **Cron 支持**：内置预设（`morning`, `twice`）或完全自定义 Cron 表达式，兼容 5 位、6 位（含秒）和 7 位（含年）格式。
*   **多时段随机触发**：支持配置多个时间段（如 08:00-10:00），系统每天自动随机抽选时间点，像真人一样不规律作息。
*   **任务持久化恢复**：所有的“定时随机延迟”均写入 SQLite 数据库。即使 Bot 重启、断网或崩溃，也能在启动时精准恢复未执行的分享任务，**彻底告别重启导致的漏发或连发**。

## 🚀 快速开始

1. **安装并启用插件**
   - 在 AstrBot 插件市场搜索 `astrbot_plugin_daily_sharing` 进行安装使用。
   - 如果要使用配图、视频、QQ空间或长效记忆，建议同时安装上方推荐插件。

2. **确认管理员权限**
   - `/分享` 系列指令默认需要 AstrBot 管理员权限。
   - 如果出现“权限不足”，请把当前会话的 `UID` 加入管理员列表（AstrBot配置 - 普通配置 - 平台配置 - 管理员ID）。
   - `UID` 可通过 `/sid` 获取。

3. **获取目标 ID**
   - 在目标私聊或群聊发送 `/sid`。
   - 私聊复制 `UID` 或 `Session ID`；群聊复制 `Session ID`。
   - 只填写纯 ID 即可。

4. **添加接收对象**
   - 在当前会话执行 `/分享 添加当前`，加入每日分享接收对象。
   - 在当前会话执行 `/分享 添加当前 早报`，加入定时早报接收对象。
   - 也可以直接在 WebUI 配置里填写接收对象 ID。

5. **手动验证**
   - 执行 `/分享 状态` 查看配置和定时状态。
   - 执行 `/分享 心情` 或 `/分享 新闻` 测试当前会话发送链路。
   - 执行 `/分享 60s` 或 `/分享 ai` 测试早报接口。

## 🧭 平台能力对照

| 平台 | 支持场景 | 接收对象填写 | 昵称/历史能力 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| QQ / OneBot (`aiocqhttp`) | 私聊、群聊 | 私聊填 QQ 号，群聊填群 `Session ID` | 私聊会尝试读取备注/昵称和好友聊天历史 | 推荐通过 `/sid` 获取|
| 个人微信 (`weixin_oc`) | 仅个人私聊 | 填微信 openid / `Session ID` | 平台接口不稳定提供真实昵称；优先使用 AstrBot 人格设定中的用户称呼 | 不支持群聊；主动发送需要 AstrBot 已保存该 openid 的 `context_token` |
| QQ空间 | 说说分享 | 不填普通聊天接收对象 | 使用 `astrbot_plugin_qzone` 分享 | 通过“空间”指令或 QQ空间专属配置触发 |

个人微信专属逻辑只对平台名或适配器元信息精确为 `weixin_oc` 的适配器启用，不会把公众号、企业微信等其它微信系适配器自动当作个人微信处理。

## 🎯 目标 ID 与配置示例

接收对象只支持 `/sid` 输出的纯 `UID` / `Session ID`。不要填写完整 UMO。

**正确示例**

```text
123456
o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat
123456:greeting,mood
123456:0 8 * * *:greeting,mood
o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat:0 23 * * *:mood,greeting
```

**错误示例**

```text
default:FriendMessage:123456
default:FriendMessage:o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat
```

配置格式说明：

- `纯 ID`：跟随全局定时和当前时段序列。
- `纯 ID:类型序列`：跟随全局定时，但只按指定类型轮换，例如 `123456:mood,news`。
- `纯 ID:Cron:类型序列`：完全独立定时，例如 `123456:0 8 * * *:greeting,mood`。
- 支持类型：`greeting`、`news`、`mood`、`knowledge`、`recommendation`。
- Cron 可填写 5/6/7 位格式：`0 8 * * *`、`0 0 8 * * *`、`0 0 8 * * * 2026` 都能被解析。

> **💡 触发秘籍**：你可以在任何请求后面加上后缀要求，例如**“要视频哦”**、**“发一段语音”** 或 **“发条说说”**。

| 你的意图 | 示例说辞 (自然语言) | Bot的实际动作 |
| :--- | :--- | :--- |
| **发空间说说** | “今天天气不错，**发个空间说说吧**” | 生成第一人称心情独白，并附带生活照分享到 QQ 空间。 |
| **看动图/视频** | “早上好，给我**发个自拍视频**看看” | 强制触发画图+视频生成，发送一段会动的问候视频。 |

## 🎮 控制台指令 (Admin Only)

以下指令默认需要 AstrBot 管理员权限。尤其是 `/分享 添加当前`、`/分享 添加当前 早报`、`/分享 开启`、`/分享 关闭` 这类会修改配置或定时状态的操作，非管理员会被 AstrBot 拦截。

| 指令格式 | 参数示例 | 功能说明 |
| :--- | :--- | :--- |
| `/分享` | `[类型]` | 手动触发分享（默认只发给当前窗口）。 |
| `/分享` | `[类型] 广播` | 向配置中的所有群聊和私聊发送。 |
| `/分享` | `[类型] 空间` | 单独生成文案并一键分享到QQ空间。 |
| `/分享` | `新闻 [源]` | 获取指定源的热搜（文字版） |
| `/分享` | `新闻 [源] 图片` | 获取指定源的热搜长图。 |
| `/分享` | `60s` / `ai` | 立即手动发送【每天60s读懂世界】或【AI资讯快报】。 |
| `/分享` | `添加当前 [类型序列]` | 将当前会话加入每日分享接收对象。 |
| `/分享` | `添加当前 早报` | 将当前会话加入定时早报接收对象。 |
| `/分享` | `昵称 [名称]` | 为当前会话设置本地昵称映射。 |
| `/分享` | `早报空间 开启/关闭` | 切换是否将定时早报自动分享到QQ空间。 |
| `/分享` | `开启` / `关闭` | 启停所有定时自动分享任务。 |
| `/分享` | `状态` | 查看运行状态、历史记录、序列索引 |
| `/分享` | `查看序列` | 查看当前时间段会轮换发什么内容 |
| `/分享` | `指定序列 [序号]` | 手动跳到序列中的某一步 (支持加后缀 `空间`) |
| `/分享` | `重置序列` | 重置轮换顺序，回到列表开头 |

参数规则补充：`新闻` 类型支持新闻源和 `图片` 参数，例如 `/分享 新闻 微博 图片`；其它类型只支持 `广播`、`空间` 后缀。如果想指定知识或推荐主题，请通过内容库配置，或直接用自然语言让 LLM 触发工具。

## ⚙️ 进阶配置指南 (Config)

请在 AstrBot WebUI 的 **插件配置** 面板中进行调整。

<details>
<summary><b>点击展开：核心配置项说明</b></summary>

1. **🎯 千群千面推送配置**
   - 推送目标支持三段式高级语法：`纯 ID:独立定时:分享类型`。
   - 例如填写 `123456:0 7 * * *:news,mood`，意味着该群脱离全局设定，拥有独立的早7点闹钟，且只轮换发送新闻和心情。
   - 接收对象 ID 通过 `/sid` 获取：在目标私聊或群聊发送 `/sid`，私聊复制 `UID`/`Session ID`，群聊复制 `Session ID`。
   - 接收对象只填写纯 ID，不填写完整 UMO。完整 UMO 形如 `default:FriendMessage:SessionID`，会被视为无效配置。
   - 个人微信 `weixin_oc` 官方适配器仅支持个人私聊，不支持群聊；私聊配置可填写 `/sid` 输出的纯 ID，例如个人 QQ `123456`，个人微信 `o2...@im.wechat`。
   - 个人微信专属逻辑只识别精确的 `weixin_oc` 适配器；其它微信系平台不会自动套用个人微信发送和图片优化规则。
   - 微信定时发送走 AstrBot 原生 `send_by_session`，不依赖插件事件缓存；但 `weixin_oc` 需要 AstrBot 已保存该 openid 的 `context_token`。如果缺失，请让该微信私聊先给机器人发任意消息刷新。
   - 自定义用户称呼映射是可选补充：QQ/OneBot 私聊会优先通过接口获取用户称呼备注或用户昵称；个人微信 `weixin_oc` 无法从平台接口稳定获取用户昵称，但私聊称呼会优先读取 AstrBot 人格设定中的用户称呼。`/分享 昵称 用户称呼` 主要用于把生活日程/记忆里的用户称呼识别为当前私聊对象；也可在配置 `contact_aliases` 中手动维护 `UID或Session ID:用户称呼`。
2. **📚 内容库自定义**
   - 自定义知识和推荐的话题池。格式为 `大类: 标签1, 标签2`。（例如：`游戏: 独立神作, 治愈解谜`）。
   - `show_knowledge_type_prefix` 控制知识分享是否显示 `知识类型: 大类 - 子类` 前缀。
   - `show_rec_type_prefix` 控制推荐分享是否显示 `推荐类型: 大类 - 子类` 前缀。
3. **📅 早报设置**
   - 独立配置每天 `08:00` 发送的长图早报。支持一键开启分享到 QQ 空间。
   - 可配置早报定时随机延迟分钟数，到点后在 `0-N` 分钟内随机发送，并支持重启后恢复未执行的延迟早报。
4. **🌟 QQ空间专属设置**
   - 拥有完全独立于群聊/私聊的 Cron 定时器（如 `0 20 * * *`）和分享序列。
   - 自动屏蔽“大家好”等社交词汇，强制输出第一人称“日记体”。
   - 分享时如遇 QQ 空间 Cookie 过期、图片上传返回登录失效等问题，会尝试清理底层 Cookie 缓存、唤醒重新登录流程，并自动重试一次分享。
5. **🎨 配图与视频**
   - **强烈建议开启**，配合 Gitee 插件实现人设外貌的绝对稳定。
   - 启用视频后，插件会为图生视频生成独立的动态提示词与声音提示词；若 LLM 生成失败，会回退到保守的默认视频提示词。
   - QQ/OneBot 群聊和私聊发送图片时会优先下载到插件数据目录的 `Temp` 后再发送，减少远程 CDN 链接不稳定导致的整条消息发送失败。
   - 个人微信 `weixin_oc` 发送图片前会默认把适配器 API/CDN 上传超时调到 60 秒，并生成一份高质量发送副本：保留 4K 最长边，目标大小默认 10240KB，优先使用高质量 JPEG。QQ 和 QQ空间不受影响。
   - 个人微信压缩副本统一放在插件数据目录的 `Temp/weixin_send_*.jpg`，默认最多保留最新 10 张，超过后自动删除最旧文件；可通过 `weixin_temp_cleanup_max_count` 调整，设为 `0` 可关闭清理。
   - 如果个人微信图片小于 10240KB 仍上传超时，通常是 CDN 上传网络或适配器超时问题，可适当调高 `weixin_api_timeout_seconds`（个人微信发送超时）。
6. **🔍 联网检索**
   - 默认开启。新闻分享会优先使用 API 返回的摘要/正文，只有缺少背景时才调用 AstrBot 内置的 Tavily/Brave 搜索；知识和推荐分享仍会使用联网检索增强真实性和深度。
   - 搜索提供商与 Key 从 AstrBot 运行时配置读取（普通配置 - AI 配置 - 网页搜索）。
   - 新闻源解析会兼容 `description`、`summary`、`content`、`mobile_url`、`hot_value_desc` 等常见字段；API 已提供可靠背景时会跳过 Web 搜索，节省搜索额度。
   - 插件发送长图时会缓存同源 JSON，想看原文可直接自然语言询问，例如“把刚才热搜图第三条原文发我”，由 LLM 触发 `news_link` 工具；最终回复保留模型自然表达，并清理多余参考链接。
7. **🤖 模型与人设**
   - 可单独指定负责此插件的 LLM 模型。如果该模型崩溃，插件会自动降级寻找系统中的第一个可用模型。

## 🧯 FAQ / 常见排障

### Q1：为什么手动 `/分享 新闻` 能发，定时任务却发不了个人微信？

手动触发时 AstrBot 当前会话事件是现成的；定时任务属于主动发送，需要通过 `send_by_session` 找到对应平台会话。对 `weixin_oc` 来说，AstrBot 还需要保存过该 openid 的 `context_token`。如果日志提示缺少或未发现 `context_token`，请让该微信私聊先给机器人发任意消息，再等待下一次定时任务或手动执行一次分享。

### Q2：`请先在对应微信私聊里发一条消息` 是什么意思？

这不是插件自己的缓存，也不是聊天历史。它指的是 `weixin_oc` 适配器主动发送消息所需的内部凭据。微信用户给机器人发过消息后，AstrBot 会保存 `weixin_oc_context_tokens`，之后定时任务才更容易主动发回该微信私聊。

### Q3：个人微信支持群聊吗？

不支持。AstrBot 的 `weixin_oc` 个人微信适配器基于 openclaw-weixin 接口，目前仅支持个人私聊，一号一人使用。群聊目标请使用 QQ/OneBot 或其它支持群聊的平台。

### Q4：为什么配置完整 UMO 会被跳过？

本插件接收对象只使用 `/sid` 输出的纯 `UID` / `Session ID`。完整 UMO 例如 `default:FriendMessage:123456` 或 `default:FriendMessage:o2...@im.wechat` 容易导致平台判断错误，因此不会作为有效目标。请改填 `123456` 或 `o2...@im.wechat`。

### Q5：为什么个人微信图片小于 10240KB 也可能上传超时？

`10240KB` 是插件压缩目标，不是微信 CDN 上传一定成功的保证。`weixin_oc` 上传还受网络、CDN、适配器超时影响。遇到 `upload_to_cdn TimeoutError` 时，可优先调高 `weixin_api_timeout_seconds`（个人微信发送超时），或降低 `weixin_image_max_size_kb`（个人微信图片目标大小）。

### Q6：个人微信压缩后的图片会一直堆积吗？

不会。发送前生成的微信压缩副本只保存在 `Temp/weixin_send_*.jpg`，插件启动、每日凌晨任务以及每次生成新副本后都会按数量清理，默认只保留最新 10 张。需要更少或更多缓存时，调整 `weixin_temp_cleanup_max_count`；设为 `0` 则关闭自动清理。

### Q7：QQ空间 Cookie 过期会不会导致定时说说一直失败？

插件会尽量兜底。分享说说时如果捕获到登录失效、图片上传接口返回异常等情况，会清理底层空间插件的 Cookie 缓存，尝试唤醒重新登录/刷新流程，并自动重试一次分享。若仍失败，通常需要检查 QQ 空间插件本身的登录状态或风控情况。

### Q8：为什么 QQ 群/私聊图片要先下载到本地？

部分 QQ 适配器处理远程 CDN 图片时容易出现 `rich media transfer failed`，严重时会导致整条消息发送失败。插件现在会优先把图片下载到 `Temp` 临时目录，再用本地图片发送，以换取更稳定的发送链路。

### Q9：为什么让 Bot 发第 5 条新闻链接时，曾经会多出“参考链接”？

这是 AstrBot 或模型在工具链最终回复阶段追加的来源引用，不一定受“显示来源引用”开关影响。插件现在会保留 LLM 自然回复，只清理末尾多余的“参考链接”列表。

### Q10：QQ 能识别昵称，个人微信为什么不一定能？

QQ/OneBot 通常能通过接口读取备注或昵称；`weixin_oc` 个人微信接口不稳定提供真实昵称。个人微信私聊称呼会优先使用 AstrBot 人格设定里的用户称呼；`/分享 昵称 称呼` 主要用于把日程、记忆里的用户称呼映射到当前私聊对象。

### Q11：为什么 `/分享 添加当前` 提示权限不足？

这是 AstrBot 权限系统拦截，不是插件发送逻辑错误。`/分享` 系列配置修改指令需要管理员权限，请通过 `/sid` 获取当前账号 `UID`，并加入 AstrBot 管理员列表。

### Q12：为什么 `/分享 推荐 动漫` 会提示无效参数？

`问候`、`心情`、`知识`、`推荐` 这类非新闻命令不把第三个词当作主题参数；它们只接受 `广播` 或 `空间` 后缀。
