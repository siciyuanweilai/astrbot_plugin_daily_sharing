## 📖 序言：打破“被动回复”的边界

这是一款为 AstrBot 打造的每日分享插件。它不再只等待一句问题才回应，而是会循着时间、天气、生活状态、聊天记忆和网络热搜，在合适的时刻把所见所闻分享到你身边。

清晨可以是一句带着阳光的早安语音，午后可以是一张热搜长图，深夜可以是一段安静的心情视频，也可以自动写成 QQ 空间里的日常说说。它关心的不只是“分享出去”，而是让每一次分享都更自然、更像正在生活。


## 🌟 核心能力 

### 1. 🔍 真实来源：联网检索增强 (Tavily / Brave)
*   **拒绝热搜标题党**：优先使用新闻源 API 自带的摘要/正文；缺少背景时再自动接入全网检索能力。分享新闻不再仅凭标题进行空洞脑补。
*   **API v2 与多源 JSON 结构兼容**：热搜接口使用新版 `/api/v2/*` 路径；解析兼容不同新闻源的返回结构与字段。
*   **百科+网页并发校验**：在进行【知识】与【推荐】时，并发调用“百度百科 + Web 搜索”，从根本上解决 LLM 胡编乱造的幻觉问题。

### 2. 🌐 跨域分享：QQ 空间与独立早报联动 
*   **说说自动机**：无需人工干预，Bot 会根据一天中的不同时段（早晨问候/深夜emo），自动以“第一人称”口吻生成配图说说并分享到QQ空间。
*   **登录态自动保活**：QQ 空间 Cookie 过期或图片上传接口返回登录失效时，会尝试唤醒底层空间插件刷新登录态并自动重试分享，降低定时说说中断概率。
*   **早报速递员**：支持将【每天 60s 读懂世界】和【AI 资讯快报】作为独立定时任务，自动推送到群聊、私聊并分享至QQ空间。

### 3. 🧠 懂得分寸：群聊气氛、聊天历史与动态记忆
*   **KY 规避 (读空气)**：群聊态势感知。当群友正在热火朝天地讨论时，Bot 会保持安静，绝不打断施法；插件自己的分享不会计入群聊热度，避免“自己把自己判断成热聊”。
*   **AstrBot 历史兜底**：OneBot 优先拉取平台深度聊天记录；非 OneBot 或平台拉取失败时，会先读取 AstrBot 保存的“平台消息记录”，如果没有可用记录，再读取 AstrBot 的“LLM 会话历史”，WebChat 等平台的上下文更稳。
*   **记忆回写 (拒绝断片)**：Bot 发送的内容会写回 AstrBot 对话历史，并用内部标记区分真实用户发言与分享。你问：“你刚才发的照片是在哪拍的？”，它能对答如流。
*   **分享背景化**：读取历史时会过滤内部触发标记，并把 Bot 之前分享过的内容降级为“背景：你之前分享过……”，不伪装成普通聊天发言。
*   **第二人称防呆**：深度优化底层提示词，彻底解决 Bot 在私聊时把用户当“第三人称”称呼的语病，交互更像真人伴侣。

### 4. 🎨 视觉与影像：形象参考与视频生成
*   **形象一致性**：联动**Gitee**生图插件，支持**形象参考图**。Bot 发的生活照终于可以拥有稳定的“脸”和气质了！
*   **图文转视频**：一键将生成的静态生活照转换为**高清动态视频**。清晨的早安，是真的在对你挥手微笑。
*   **视频提示词**：视频生成会基于配图描述和分享文案分别设计镜头动作、人物微表情、环境声与人声氛围，尽量保持原图人物、服装、构图和场景一致。
*   **构图顾问**：内置**Agent**智能判断 —— 分享问候、心情时“画人”，分享新闻、知识和推荐时“画景”。
*   **环境光影**：自动根据物理时间（清晨/黄昏/深夜）调整画面的光影，杜绝 "半夜窗外是大太阳" 的违和感。

### 5. 📰 见闻来源：全网热搜与自定义内容库
*   **众多热搜源**：微博、知乎、B站、抖音、头条、百度、腾讯、小红书、夸克、36氪、51CTO、A站、爱范儿、网易、新浪、澎湃、第一财经、财联社。
*   **长图原文反查**：新闻热搜长图会同步缓存同源 JSON，之后可用自然语言说“把刚才热搜图第三条原文发我”，由 LLM 调用工具取回对应新闻链接，并清理多余参考链接。
*   **时段偏好算法**：早晨看知乎，傍晚刷微博，深夜看抖音，完美契合人类作息。
*   **专属内容库**：支持在配置项中完全自定义【知识】和【推荐】库的话题方向，并可选择是否显示“知识类型/推荐类型”前缀。

### 6. 🛡️ 稳定送达：双重降级机制
*   **会话模型优先**：如果插件未单独指定 LLM，会优先使用 AstrBot 当前会话配置的模型；只有无法解析时才使用系统默认模型。不同群聊、私聊的模型配置可以自然生效。
*   **LLM 自动降级**：指定的大模型 API 挂了？超时了？触发敏感词了？插件会临时切换到系统默认备用模型，过期后再恢复尝试指定模型。
*   **新闻源熔断**：主新闻源获取失败时，自动在备选池中切换；全部失败时明确取消本次新闻分享并写入失败记录，避免输出无依据新闻。
*   **触发反馈可见**：通过 LLM 工具触发分享时，支持给原消息贴上“处理中 / 成功 / 失败”表情；不支持该接口的平台会静默跳过，不影响分享流程。
*   **图片发送稳态链路**：群聊、私聊图片会优先下载到本地临时目录后再发送，规避不稳定 CDN 链接导致的 `富媒体传输失败`。
*   **临时文件自清理**：普通下载图和个人微信压缩副本统一进入 `Temp` 管理；个人微信压缩副本默认只保留最新 10 张，避免长期堆积。

### 7. ⏰ 灵活调度：定时策略与断点恢复
*   **时段序列**：不同时间段分享不同内容（早晨发问候，下午发推荐，晚上发心情）。
*   **Cron 支持**：内置预设（`morning`, `twice`）或完全自定义 Cron 表达式，兼容 5 位、6 位（含秒）和 7 位（含年）格式。
*   **多时段随机触发**：支持配置多个时间段（如 08:00-10:00），系统每天自动随机抽选时间点，像真人一样不规律作息。
*   **任务持久化恢复**：所有的“定时随机延迟”均写入 SQLite 数据库。即使 Bot 重启、断网或崩溃，也能在启动时精准恢复未完成的分享任务，**彻底告别重启导致的漏发或连发**。

### 8. 🖥️ 可视化管理：仪表盘与设置页
*   **仪表盘状态栏**：集中展示自动分享、当前时段、接收目标、定时任务、文案/图片/视频数量、分享进度等状态。
*   **设置页覆盖 `_conf_schema.json`**：所有配置集中在仪表盘的设置页中调整，支持自动保存并在刷新后保持当前页面。
*   **甜妹风交互**：樱花灯笼开关、梦幻鼠标、加载动画、滑块花瓣、媒体高光和轻量动效都在前端内置，不影响分享主流程。

## 🚀 快速开始

1. **安装并启用插件**
   - 在 AstrBot 插件市场搜索 `astrbot_plugin_daily_sharing` 进行安装使用。
   - 如果要使用配图、视频、QQ空间或长效记忆，建议同时安装上方推荐插件。

2. **确认管理员权限**
   - `/分享` 系列指令默认需要 AstrBot 管理员权限。
   - 如果出现“权限不足”，请把当前会话的 `UID` 加入管理员列表（AstrBot配置 - 普通配置 - 平台配置 - 管理员ID）。
   - `UID` 可通过 `/sid` 获取。

3. **获取目标 ID**
   - 在目标私聊或群聊发送 `/sid`。
   - 私聊复制 `UID` 或 `Session ID`；群聊复制 `Session ID`。
   - 只填写纯 ID 即可。

4. **添加接收对象**
   - 在当前会话发送 `/分享 添加当前`，加入每日分享接收对象。
   - 在当前会话发送 `/分享 添加当前 早报`，加入定时早报接收对象。
   - 也可以直接在 WebUI 配置里填写接收对象 ID。

5. **手动验证**
   - 发送 `/分享 状态` 查看配置和定时状态。
   - 发送 `/分享 心情` 或 `/分享 新闻` 测试当前会话发送链路。
   - 发送 `/分享 60s` 或 `/分享 ai` 测试早报接口。

6. **打开 Dashboard**
   - 在 AstrBot WebUI 的插件页面进入本插件 Dashboard。
   - 仪表盘可查看状态、动态、媒体、日历和生成进度；设置页可直接调整 `_conf_schema.json` 中的配置并自动保存。

## 🧭 平台能力对照

| 平台 | 支持场景 | 接收对象填写 | 昵称/历史能力 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| QQ / OneBot (`aiocqhttp`) | 私聊、群聊 | 私聊填 QQ 号，群聊填群 `Session ID` | 私聊会尝试读取备注/昵称和好友聊天历史 | 推荐通过 `/sid` 获取|
| 个人微信 (`weixin_oc`) | 仅个人私聊 | 填微信 openid / `Session ID` | 平台接口不稳定提供真实昵称；优先使用 AstrBot 人格设定中的用户称呼 | 不支持群聊；主动发送需要 AstrBot 已保存该 openid 的 `context_token` |
| WebChat / 其它 AstrBot 平台 | 取决于平台适配器 | 优先使用 `/sid` 输出 | 优先读取 AstrBot 保存的平台消息记录 | 主动发送是否可用取决于适配器是否支持 `send_by_session` |
| QQ空间 | 说说分享 | 不填普通聊天接收对象 | 使用 `astrbot_plugin_qzone` 分享 | 通过“空间”指令或 QQ空间专属配置触发 |

个人微信专属逻辑只对平台名或适配器元信息精确为 `weixin_oc` 的适配器启用，不会把公众号、企业微信等其它微信系适配器自动当作个人微信处理。

## 🎯 目标 ID 与配置示例

接收对象只支持 `/sid` 输出的纯 `UID` / `Session ID`。不要填写完整 UMO。

**正确示例**

```text
123456
o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat
123456:greeting,mood
123456:0 8 * * *:greeting,mood
o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat:0 23 * * *:mood,greeting
```

**错误示例**

```text
default:FriendMessage:123456
default:FriendMessage:o2cq508wSbYFvbrV5tnC29LoU3NU@im.wechat
```

配置格式说明：

- `纯 ID`：跟随全局定时和当前时段序列。
- `纯 ID:类型序列`：跟随全局定时，但只按指定类型轮换，例如 `123456:mood,news`。
- `纯 ID:Cron:类型序列`：完全独立定时，例如 `123456:0 8 * * *:greeting,mood`。
- 支持类型：`greeting`、`news`、`mood`、`knowledge`、`recommendation`。
- Cron 可填写 5/6/7 位格式：`0 8 * * *`、`0 0 8 * * *`、`0 0 8 * * * 2026` 都能被解析。

## 💬 沉浸式自然语言交互

得益于**AstrBot**的工具函数调用能力，**无需记忆繁琐指令**，直接和Bot聊天即可触发分享！

> **💡 触发秘籍**：你可以在任何请求后面加上后缀要求，例如**“要视频哦”**、**“发一段语音”** 或 **“发条说说”**。
>
> 前提：AstrBot 需要启用 LLM 工具调用；当前会话需是管理员或已配置接收对象。QQ 空间、视频、语音与早报图片还需要对应插件/API/配置开关可用。
> 支持消息贴表情接口的平台会自动标记分享状态；如果平台不支持，分享仍会正常发送。

| 你的意图 | 示例说辞 (自然语言) | Bot的实际动作 |
| :--- | :--- | :--- |
| **发空间说说** | “今天天气不错，**分享下空间说说吧**” | 调用 QQ 空间分享流程，按空间独立序列或指定类型生成第一人称动态；开启配图时附带生活照。 |
| **看图片视频** | “早上好，给我**分享下心情视频**看看” | 触发配图与图生视频流程；在 AI 配图、视频开关和图生视频插件可用时发送动态视频。 |
| **看独立早报** | “给我来一份**60s新闻**” 、 “看看**AI资讯**” | 直接获取当天的 60s 读懂世界 或 AI资讯快报 长图并发送。 |
| **吃瓜看热搜** | “看看**微博热搜**的图片版” | 获取微博实时热搜长图，并缓存同源新闻列表用于后续查链接。 |
| **查原文链接** | “刚才热搜图**第3条链接**发我” | 读取最近缓存的同源热搜列表，由 LLM 调用 `news_link` 返回对应原文链接，配置柠柚 API Key 后优先返回短链接。 |
| **听碎碎念** | “你现在心情怎么样？**分享下语音**” | 结合它此时的“虚拟日程”生成内容；开启 TTS 时额外发送语音。 |

还可以这样说：

- “来个**文字版微博热搜**” 、“看热搜，**不要图片**”：生成文字版新闻分享。
- “发个知识，**配张图**” 、 “讲个小知识，**念给我听**”：分别触发配图或语音。
- “**第十二条网址**” 、“**空间那条第3条链接**”：从最近缓存或 QQ 空间新闻缓存里反查原文，支持序号、标题关键词和最近关注项。
- “把**60s新闻发空间**”、“**AI资讯分享到空间**”：管理员可直接把早报长图同步到 QQ 空间。

## 🎮 控制台指令 (Admin Only)

以下指令默认需要 AstrBot 管理员权限。尤其是 `/分享 添加当前`、`/分享 添加当前 早报`、`/分享 开启`、`/分享 关闭` 这类会修改配置或定时状态的操作，非管理员会被 AstrBot 拦截。

| 指令格式 | 参数示例 | 功能说明 |
| :--- | :--- | :--- |
| `/分享` | `[类型]` | 手动触发分享（默认只发给当前窗口）。 |
| `/分享` | `[类型] 广播` | 向配置中的所有群聊和私聊发送。 |
| `/分享` | `[类型] 空间` | 单独生成文案并一键分享到QQ空间。 |
| `/分享` | `新闻 [源]` | 获取指定源的热搜（文字版） |
| `/分享` | `新闻 [源] 图片` | 获取指定源的热搜长图。 |
| `/分享` | `60s` / `ai` | 立即手动发送【每天60s读懂世界】或【AI资讯快报】。 |
| `/分享` | `添加当前 [类型序列]` | 将当前会话加入每日分享接收对象。 |
| `/分享` | `添加当前 早报` | 将当前会话加入定时早报接收对象。 |
| `/分享` | `昵称 [名称]` | 为当前会话设置本地昵称映射。 |
| `/分享` | `早报空间 开启/关闭` | 切换是否将定时早报自动分享到QQ空间。 |
| `/分享` | `开启` / `关闭` | 启停所有定时自动分享任务。 |
| `/分享` | `状态` | 查看运行状态、历史记录、序列索引 |
| `/分享` | `查看序列` | 查看当前时间段会轮换发什么内容 |
| `/分享` | `指定序列 [序号]` | 手动跳到序列中的某一步 (支持加后缀 `空间`) |
| `/分享` | `重置序列` | 重置轮换顺序，回到列表开头 |

参数规则补充：`新闻` 类型支持新闻源和 `图片` 参数，例如 `/分享 新闻 微博 图片`；其它类型只支持 `广播`、`空间` 后缀。如果想指定知识或推荐主题，请通过内容库配置，或直接用自然语言让 LLM 触发工具。

- **状态总览**：查看自动分享开关、全局触发、QQ 空间、早报、当前时段、定时任务、接收目标和分享进度。
- **手动触发**：在页面内选择群聊、私聊、QQ 空间或早报，并指定分享类型、新闻源；选择仅群聊/仅私聊时可填写一个指定群号或 QQ 号，留空则按原逻辑发送全部目标。
- **生成进度**：分享过程中显示“文案生成中、配图生成中、视频生成中、语音生成中、发送中”，失败的多媒体阶段会标记为跳过或降级，不影响已生成内容继续发送。
- **动态与媒体**：展示今日分享、文案数量、图片数量、视频数量等统计；动态记录、图片、视频和文案支持页面内查看。
- **媒体预览**：图片和视频支持点击打开、再次点击关闭；视频控制条包含播放/暂停、进度条和静音，移动端可先唤出控制条再拖动进度。
- **日历轮播**：定时任务会按日期分组自动轮播，移动端保持固定高度，避免任务少时面板收缩。
- **设置入口**：设置页与仪表盘在同一个 Dashboard 内切换，页面会记住当前视图；刷新时会显示“少女祈祷中...”加载动画。
- **页面偏好**：樱花飘落开关会保存到 `dashboard_preferences.json`，关闭后刷新或重新打开 Dashboard 仍保持关闭。

页面操作会调用插件内置的 `page/status`、`page/config`、`page/preferences`、`page/history`、`page/media`、`page/toggle`、`page/run`、`page/retry`、`page/targets` Web API；当前前端以状态轮询刷新为主，分享中会加快轮询频率。如果页面提示没有检测到 AstrBot Pages bridge，请从 AstrBot WebUI 的插件页面入口打开。

## 🧯 FAQ / 常见排障

### Q1：为什么手动 `/分享 新闻` 能发，定时任务却发不了个人微信？

手动触发时 AstrBot 当前会话事件是现成的；定时任务属于主动发送，需要通过 `send_by_session` 找到对应平台会话。对 `weixin_oc` 来说，AstrBot 还需要保存过该 openid 的 `context_token`。如果日志提示缺少或未发现 `context_token`，请让该微信私聊先给机器人发任意消息，再等待下一次定时任务或手动分享一次。

### Q2：`请先在对应微信私聊里发一条消息` 是什么意思？

这不是插件自己的缓存，也不是聊天历史。它指的是 `weixin_oc` 适配器主动发送消息所需的内部凭据。微信用户给机器人发过消息后，AstrBot 会保存 `weixin_oc_context_tokens`，之后定时任务才更容易主动发回该微信私聊。

### Q3：个人微信支持群聊吗？

不支持。AstrBot 的 `weixin_oc` 个人微信适配器基于 openclaw-weixin 接口，目前仅支持个人私聊，一号一人使用。群聊目标请使用 QQ/OneBot 或其它支持群聊的平台。

### Q4：为什么配置完整 UMO 会被跳过？

本插件接收对象只使用 `/sid` 输出的纯 `UID` / `Session ID`。完整 UMO 例如 `default:FriendMessage:123456` 或 `default:FriendMessage:o2...@im.wechat` 容易导致平台判断错误，因此不会作为有效目标。请改填 `123456` 或 `o2...@im.wechat`。

### Q5：为什么个人微信图片小于 10240KB 也可能上传超时？

`10240KB` 是插件压缩目标，不是微信 CDN 上传一定成功的保证。`weixin_oc` 上传还受网络、CDN、适配器超时影响。遇到 `upload_to_cdn TimeoutError` 时，可优先调高 `weixin_api_timeout_seconds`（个人微信发送超时），或降低 `weixin_image_max_size_kb`（个人微信图片目标大小）。

### Q6：个人微信压缩后的图片会一直堆积吗？

不会。发送前生成的微信压缩副本只保存在 `Temp/weixin_send_*.jpg`，插件启动、每日凌晨任务以及每次生成新副本后都会按数量清理，默认只保留最新 10 张。需要更少或更多缓存时，调整 `weixin_temp_cleanup_max_count`；设为 `0` 则关闭自动清理。

### Q7：QQ空间 Cookie 过期会不会导致定时说说一直失败？

插件会尽量兜底。分享说说时如果捕获到登录失效、图片上传接口返回异常等情况，会清理底层空间插件的 Cookie 缓存，尝试唤醒重新登录/刷新流程，并自动重试一次分享。若仍失败，通常需要检查 QQ 空间插件本身的登录状态或风控情况。

### Q8：为什么 QQ 群/私聊图片要先下载到本地？

部分 QQ 适配器处理远程 CDN 图片时容易出现 `富媒体传输失败`，严重时会导致整条消息发送失败。插件现在会优先把图片下载到 `Temp` 临时目录，再用本地图片发送，以换取更稳定的发送链路。

### Q9：为什么让 Bot 发第 5 条新闻链接时，曾经会多出“参考链接”？

这是 AstrBot 或模型在工具链最终回复阶段追加的来源引用，不一定受“显示来源引用”开关影响。插件现在会保留 LLM 自然回复，只清理末尾多余的“参考链接”列表。

### Q10：QQ 能识别昵称，个人微信为什么不一定能？

QQ/OneBot 通常能通过接口读取备注或昵称；`weixin_oc` 个人微信接口不稳定提供真实昵称。个人微信私聊称呼会优先使用 AstrBot 人格设定里的用户称呼；`/分享 昵称 称呼` 主要用于把日程、记忆里的用户称呼映射到当前私聊对象。

### Q11：为什么 `/分享 添加当前` 提示权限不足？

这是 AstrBot 权限系统拦截，不是插件发送逻辑错误。`/分享` 系列配置修改指令需要管理员权限，请通过 `/sid` 获取当前账号 `UID`，并加入 AstrBot 管理员列表。

### Q12：为什么 `/分享 推荐 动漫` 会提示无效参数？

`问候`、`心情`、`知识`、`推荐` 这类非新闻命令不把第三个词当作主题参数；它们只接受 `广播` 或 `空间` 后缀。

### Q13：分享写入聊天历史，会污染后续对话吗？

不会按普通对话污染。插件写入历史时会用 `愿此见闻悄然为我启封` 写入一条内部触发；下一条 assistant 只保存正文，不再追加分享前缀。读取历史时，内部触发会被过滤；紧随其后的 assistant 正文只作为“背景：你之前分享过……”进入提示词，方便 Bot 记得自己发过什么，但不会假装用户真的要求过这次分享。写入 Memos 时也只使用 `每日分享记录` 作为中性记录标题。

### Q14：为什么插件不指定 LLM 时，每个会话可能使用不同模型？

这是为了对齐 AstrBot 的会话级模型配置。插件没有单独配置 `llm_provider_id` 时，会优先读取当前目标 UMO 对应的聊天 Provider；只有读取不到时才使用系统默认模型。

### Q15：为什么自然语言触发后没有出现处理状态表情？

状态表情依赖平台 Bot 暴露 `set_msg_emoji_like` 接口，并且原消息需要能读取到 `message_id`。如果当前平台、适配器或消息类型不支持，插件会跳过表情反馈；这只影响可视化提示，不影响实际分享、发送和失败回报。

### Q16：为什么 Dashboard 设置页没有单独做成 `pages/settings`？

AstrBot Pages 支持一个插件注册多个页面，但列表里会出现多个入口。现版把仪表盘和设置页放在同一个 `pages/dashboard` 里，是为了只保留一个入口，并让状态、手动分享、媒体和设置共享同一套页面状态。设置页刷新后会通过页面偏好回到设置视图。

### Q17：Dashboard 为什么还是轮询刷新，不直接写 SSE / WebSocket？

现版前端以 `page/status` 轮询为主：空闲时低频刷新，分享中高频刷新。这样实现简单稳定，也更兼容 AstrBot Pages bridge。代码中保留了事件通道方向，但当前正式页面不要依赖 SSE；如果后续 AstrBot 页面桥接环境更稳定，可以再把事件推送接上。
